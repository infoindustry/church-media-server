# Church Local Media Server v0.2

Локальная система управления церковным экраном: **мини-ПК + ТВ + админка с телефона**. Работает в локальной сети и рассчитана на служение без зависимости от YouTube, рекламы и интернета.

## Что есть в этой версии

- TV Screen: `/screen/main`
- Admin Panel: `/admin`
- Локальный каталог песен
- Загрузка MP4/WebM/MOV файлов
- Запуск локального видео на ТВ
- Команды видео: Play / Pause / Stop / Restart
- **План служения**: песни, Писание, QR-перевод, объявления, blank
- Кнопки **Следующий / Назад** для плана служения
- Быстрый вывод места Писания вручную
- Добавление Писания в план служения
- QR-код для live translation
- Добавление QR-перевода в план служения
- Объявления и добавление объявлений в план
- Blank screen / черный экран / заставка
- Проверка готовности перед служением
- Realtime через SSE
- JSON-хранилище без внешней базы
- Работа в локальной сети без интернета

## Быстрый запуск на мини-ПК

```bash
npm install
npm run build
npm start
```

Откройте на мини-ПК/ТВ:

```txt
http://localhost:4000/screen/main
```

С телефона в той же Wi‑Fi сети откройте:

```txt
http://IP-МИНИ-ПК:4000/admin
```

Например:

```txt
http://192.168.1.50:4000/admin
```

## Режим разработки

```bash
npm install
npm run dev
```

- Клиент: `http://localhost:5173/admin`
- Экран: `http://localhost:5173/screen/main`
- API: `http://localhost:4000/api/...`

## Как использовать на служении

1. Подключите мини-ПК к ТВ через HDMI.
2. Запустите сервер.
3. Откройте `/screen/main` на ТВ в fullscreen.
4. С телефона откройте `/admin`.
5. Добавьте локальные видео-песни.
6. В разделе **Песни** добавьте нужные песни в план.
7. В разделе **Писание** добавьте нужные места Писания в план.
8. В разделе **Перевод** добавьте QR live translation в план.
9. В разделе **План** запускайте пункты или используйте **Следующий / Назад**.
10. Перед служением откройте **Проверка** и убедитесь, что локальные файлы найдены.

## Где лежат данные

```txt
server/data/store.json
server/media/
```

`store.json` — песни, объявления, текущий экран и план служения.  
`server/media/` — загруженные видео и файлы.

## Важно про YouTube

YouTube не является основой системы. Его можно хранить как ссылку-источник, но на служении безопаснее использовать локальный MP4-файл. Если песня без локального файла, система покажет предупреждение.

## Фонограммы / локальные аудиофайлы

В версии v0.5 добавлен раздел **Фонограммы**.

Поддерживаются локальные аудиофайлы:

- MP3
- WAV
- OGG
- M4A
- AAC
- FLAC
- WebM audio

Сценарии:

1. Открыть `/admin`.
2. Перейти в раздел **Фонограммы**.
3. Загрузить MP3/WAV/другой аудиофайл.
4. Нажать **На ТВ** — на экране появится аудио-заставка, звук должен стартовать автоматически.
5. Нажать **В план** — фонограмма попадет в порядок служения.

Локальные фонограммы работают без интернета и не зависят от YouTube.

## Гостевая YouTube-ссылка: только звук

В разделе **Песни** блок “Гость принес YouTube-ссылку” теперь имеет галочку:

```txt
Только звук / без видео на экране
```

Если включить эту галочку, ТВ покажет темную аудио-заставку и попробует автоматически запустить YouTube в фоне.

Важно: YouTube остается онлайн-режимом. Возможны реклама, зависимость от интернета и ограничения автозапуска со звуком со стороны браузера.

Для надежного служения лучше заранее скачать/подготовить локальный MP3 или MP4 и использовать его из локального каталога.

## Автозапуск звука на мини-ПК

Локальные MP4/MP3 запускаются с `autoplay`, но некоторые браузеры могут блокировать автозапуск со звуком.

Для церковного мини-ПК лучше запускать Chrome/Chromium в kiosk-режиме с флагом:

```bash
--autoplay-policy=no-user-gesture-required
```

Пример для Linux:

```bash
chromium-browser --kiosk --autoplay-policy=no-user-gesture-required http://localhost:4000/screen/main
```

Пример для Windows ярлыка Chrome:

```txt
chrome.exe --kiosk --autoplay-policy=no-user-gesture-required http://localhost:4000/screen/main
```

## Serbian / Crnogorski Bible import

Чтобы добавить сербский/черногорский текст Библии для галочки **Srpski/Crnogorski**, запусти на машине с интернетом:

```bash
npm run import:serbian
```

После этого перезапусти сервер:

```bash
npm start
```

В админке открой **Писание** и выбирай: русский, English, Srpski/Crnogorski или все вместе.

Если API найдёт несколько сербских переводов, можно выбрать конкретный ID вручную:

```bash
SERBIAN_TRANSLATION_ID=TRANSLATION_ID npm run import:serbian
```

Подробности и альтернативные источники: `vendor/bibles/SERBIAN_SOURCE.md`.
